Noobcash

Noobcash is a simplified blockchain system implemented to facilitate transactions between distributed nodes while ensuring consensus using Proof-of-Work.

The project consists of a backend, responsible for crucial blockchain functionalities such as mining and block creation, and a client, which enables users to interact with the blockchain by creating transactions. The system is implemented in Python and utilizes RESTful APIs for node communication.

Key features include:

  • Genesis Block Creation: Initialized by a bootstrap node, providing initial tokens to the network.
  • Transaction Management: Each node maintains its blockchain and processes transactions in a distributed manner, achieving consensus through mining.
  • Mining Process: Transactions are collected in blocks, which are then mined using a nonce to achieve the target difficulty, ensuring the integrity of the blockchain.

Performance evaluation was conducted using virtual machines, focusing on metrics such as throughput and block time under varying block capacities and mining difficulties. Results showed the impact of increasing block capacity and mining difficulty on the overall system performance, highlighting the scalability potential of the system.

Database System

This Project focuses on the development of a relational database for managing NFC-enabled access to rooms and services at a large hotel.

The project involved designing and implementing a full database system that tracks customer access and service usage while minimizing physical interactions through digital transformation, especially important during COVID-19. The system records check-ins, room access, and services used by customers through NFC-enabled bracelets.

Key features:

  • Entity-Relationship Diagram: Designed an ER diagram to model hotel services, customer data, room access, and the NFC system.
  • Database Implementation: Developed the database using Microsoft SQL Server, with various constraints ensuring data integrity and correct relationships between entities.
  • User Access and Permissions: NFC bracelets track guest movements and access permissions, and the database records this information securely.
  • Indexes and Performance: Created indexes on frequently queried fields to enhance database performance.

The database integrates with a front-end application developed in PHP and HTML for an easy-to-use customer management interface, using XAMPP for hosting.
